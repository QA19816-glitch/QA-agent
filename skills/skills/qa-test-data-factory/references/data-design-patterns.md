# Test data patterns

## Validation data
- Required vs optional
- Format-specific valid and invalid values
- Min/max/over-limit values

## Permission data
- Admin / operator / viewer / external user
- Same-tenant vs cross-tenant user
- Owner vs non-owner

## State data
- Draft / active / pending / approved / rejected / cancelled / expired
- Data that can transition vs terminal data

## Integration data
- Normal callback payload
- Missing field payload
- Duplicate callback payload
- Delayed callback payload
- Out-of-order callback payload
