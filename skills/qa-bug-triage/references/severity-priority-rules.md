# Severity and priority rules

## Severity
- S0: system unusable, security/data-loss, or total release blocker
- S1: core feature broken with no acceptable workaround
- S2: important but partial impact, workaround may exist
- S3: minor impact or cosmetic issue

## Priority
- P0: must fix immediately for release/business continuity
- P1: should fix in current cycle
- P2: can schedule normally
- P3: backlog / low urgency

## Decision prompts
- How many users are affected?
- Is core revenue or approval flow blocked?
- Is there a safe workaround?
- Is data integrity at risk?
